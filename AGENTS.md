# Инструкции для агента

## Коммиты

**Всегда** генерируй сообщения коммитов так:

1. **Язык**: только **русский** (заголовок и тело).
2. **Формат**: Conventional Commits — `<type>[scope]: <описание>`.
3. **Типы**: `feat`, `fix`, `docs`, `style`, `refactor`, `perf`, `test`, `build`, `ci`, `chore`.
4. **Описание**: императив, до 72 символов, без точки в конце («добавить», «исправить», не «добавлено»).

Примеры:
- `chore(rules): добавить правило коммитов на русском и Conventional Commits`
- `feat(canvas): добавить экспорт карусели в PNG`
- `fix(styles): исправить отступы в мобильной версии`

Перед предложением коммита смотри реальный дифф: описание должно соответствовать изменённым файлам, а не общим формулировкам.

Подробности — в `.cursor/rules/commit-conventions.mdc`.
